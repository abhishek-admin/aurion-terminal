## WHAT IS AURION

Aurion is a **Bloomberg Terminal-inspired** Indian stock market intelligence dashboard. It runs locally from a single `python main.py` command. No accounts, no cloud deployment, no API keys required for core functionality.

**Name etymology:** "Aurion" from Latin "aurum" (gold) — modern, fictional, premium, financial.

**Core thesis:** Give retail Indian investors a single screen showing every factor that moves NIFTY — global and domestic — classified by AI sentiment, with real-time visual feedback that makes the dashboard feel alive.

### Market Stress Index (0–100)

**Weighted composite formula:**

```
Base = 40.0 (neutral)

VIX component (weight: ~25%):
  VIX > 35 → +15    VIX > 30 → +12    VIX > 25 → +8
  VIX > 20 → +4     VIX < 14 → -8     VIX < 18 → -4

INR component (weight: ~15%):
  clamp(INR_change% × 8, -10, +10)    // INR weakening = stress up

Crude component (weight: ~15%):
  clamp(BRENT_change% × 3, -8, +8)    // Oil up = stress up for India

DXY component (weight: ~10%):
  clamp(DXY_change% × 4, -6, +6)      // Strong dollar = stress up

NIFTY component (weight: ~20%):
  clamp(NIFTY_change% × 5, -12, +12)  // NIFTY up = stress DOWN (inverted)

US 10Y component (weight: ~15%):
  US10Y > 5.0% → +8    US10Y > 4.5% → +4    US10Y < 3.5% → -4

Final = clamp(sum, 0, 100), rounded to 1 decimal
```

**Spike detection:** If stress changes by ≥4 points between polls, trigger visual shiver + flash animation.

### Design Philosophy: Bloomberg Terminal

The UI must look and feel like a Bloomberg Terminal. This means:

1. **Information density** — every pixel shows data. No whitespace waste.
2. **Monospace font** (JetBrains Mono) for all data values, labels, badges
3. **Dark background** — near-black (#04070c), not grey
4. **Color language:**
   - Green (#00e68a) = bullish / positive / up / safe
   - Red (#ff4757) = bearish / negative / down / danger
   - Amber (#ffc048) = neutral / caution / awaiting
   - Cyan (#1de9b6) = highlighted / special (India 10Y)
   - Blue (#3a8bff) = links / info signals
5. **Grid layout** — 3 columns, 2 rows, separated by 1px borders (like Bloomberg panels)
6. **No rounded corners** on panels (terminal feel), minimal rounding (3-4px) on cards
7. **No shadows, no gradients** on panels — flat Bloomberg aesthetic
8. **Data updates visually** — numbers change, colors shift, gauge moves

### CRITICAL ANIMATION: Bloomberg-Style Live Ticker Blink

**This is the signature visual effect.** Every data cell that is LIVE (its market is currently open) must have a continuous blink animation that mimics a Bloomberg terminal's live price updates:

#### How It Works

When a ticker's market is OPEN, its value cell in the 20-Factor Radar (and in the topbar ticker strip) must do this continuous cycle:

1. **Phase A (0.0s–0.4s):** Background is amber/yellow (#ffc048 at 15% opacity), text is normal color
2. **Phase B (0.4s–0.8s):** Background fades to white/light (#ffffff at 8% opacity), text stays
3. **Phase C (0.8s–1.2s):** Background fades back to dark (normal tile background), text stays
4. **Loop:** Repeat from Phase A

#### CSS Implementation

```css
/* Bloomberg-style live blink for 20-Factor Radar tiles */
@keyframes live-blink {
    0%, 100% { background-color: rgba(255, 192, 72, 0.12); }   /* amber glow */
    40%      { background-color: rgba(255, 255, 255, 0.06); }   /* white flash */
    70%      { background-color: transparent; }                   /* back to base */
}

/* Apply to any heatmap tile that is LIVE */
.hc.is-live {
    animation: live-blink 2.5s ease-in-out infinite;
}

/* The value text inside a live tile subtly pulses */
.hc.is-live .hc-v {
    animation: val-pulse 2.5s ease-in-out infinite;
}

@keyframes val-pulse {
    0%, 100% { opacity: 1; }
    50%      { opacity: 0.7; }

```

#### JavaScript: Determining Which Tiles Are Live

```javascript
// After building heatmap, mark tiles as live based on market hours
function markLiveTiles() {
    const liveKeys = ['DXY', 'BRENT', 'US10Y', 'VIX_US', 'GOLD', 'NASDAQ', 'INR/USD'];
    document.querySelectorAll('.hc').forEach(tile => {
        const label = tile.querySelector('.hc-l')?.textContent?.trim();
        const key = mapLabelToTicker(label);  // e.g., 'DXY' → 'DXY'
        if (key && isMarketOpen(key)) {
            tile.classList.add('is-live');
        } else {
            tile.classList.remove('is-live');
        }
    });
}
// Call markLiveTiles() after every heatmap render and every 60 seconds
```

**The effect should be subtle but continuous — like watching a Bloomberg terminal where live cells have a barely-perceptible glow cycling. Not flashy. Comfortable. The user's eye naturally gravitates to the blinking tiles because those are the ones with live data.**

### Simulated Price Micro-Fluctuation

Between API polls (every 18s), tickers with open markets show micro-fluctuations to create a "live" feel:

```javascript
// Every 350ms for open-market tickers:
function tickPrices() {
    for (const key of Object.keys(simPrices)) {
        if (!simPrices[key].open || simPrices[key].base === 0) continue;
        // Jitter: ±0.01% of base price
        const jitter = simPrices[key].base * (Math.random() - 0.5) * 0.0002;
        simPrices[key].display = simPrices[key].base + jitter;
    }
    renderTickerStrip();
}
```

**When next API response arrives:** `simPrices[key].base` snaps to new real value. Micro-jitter continues around new base.

## RISK REGISTER

| Risk | Probability | Impact | Mitigation |
|------|-------------|--------|------------|
| yfinance rate-limited / blocked | Medium | High | Reduce poll frequency, add fallback delay, cache aggressively |
| RSS feeds return empty | Medium | Low | 15 feeds = redundancy. If one fails, 14 others still work |
| NSE symbols change | Low | Medium | Periodic manual verification of ^NSEI, ^NSEBANK, ^CNXIT |
| LLM hallucinating stock predictions | High (if enabled) | Critical | Strict prompt constraints + disclaimer UI |
| SEBI regulatory concerns | Low | Critical | "Not investment advice" disclaimer on every AI output |
| User API key theft via XSS | Low | High | No inline scripts, CSP headers in production, localStorage only |
| Data staleness undetected | Medium | Medium | Show "last refreshed Xs ago" prominently, stale indicators in statusbar |
